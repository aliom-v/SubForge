# SubForge 第六批自动化验证执行清单

## 1. 文档目的

本文档用于把第六批测试范围固定为“带变更写入的更长链路 Worker 集成测试”，避免继续扩展到全量 CRUD 或真实部署环境。

本轮只做三类验证：

1. 用户写入链路对缓存与后续订阅读取的影响
2. 节点写入链路对缓存与后续订阅读取的影响
3. 模板写入链路对缓存与后续订阅读取的影响

## 2. 本轮边界

### 2.1 本轮要补

- `apps/worker/src/index.ts` 中用户、节点、模板写入接口的请求级行为
- 写入成功后缓存失效、审计写入与后续预览 / 公开订阅读取的一致性

### 2.2 本轮不补

- 用户 / 模板 / 规则源删除能力补齐
- 真实 D1 / KV / Cloudflare 集成测试
- Web 页面级测试
- 全量写接口矩阵覆盖
- 覆盖率门槛与报告平台

## 3. 具体执行项

### 3.1 用户写入链路

目标文件：

- `apps/worker/src/index.ts`
- `apps/worker/src/repository.ts`

本轮至少覆盖：

- `POST /api/users/:id/reset-token` 会失效旧 token 与新 token 相关缓存，并使旧订阅失效、新订阅可继续访问
- `POST /api/users/:id/nodes` 更新节点绑定后，会失效该用户缓存，且后续预览 / 订阅读取反映新绑定结果
- 用户写入链路会写入审计日志，且敏感字段不直接落审计 payload

### 3.2 节点写入链路

目标文件：

- `apps/worker/src/index.ts`
- `apps/worker/src/repository.ts`

本轮至少覆盖：

- `PATCH /api/nodes/:id` 会失效受影响用户缓存，且后续订阅内容反映节点更新
- `DELETE /api/nodes/:id` 会失效受影响用户缓存，且后续订阅请求按当前实现返回无可用节点错误
- 节点写入链路会写入对应审计日志

### 3.3 模板写入链路

目标文件：

- `apps/worker/src/index.ts`
- `apps/worker/src/repository.ts`

本轮至少覆盖：

- `POST /api/templates` 在创建新的默认模板时，会按目标类型失效缓存，且后续订阅内容反映新模板
- 必要时补 `POST /api/templates/:id/set-default` 的请求级链路，确认写入、缓存失效与后续读取一致
- 模板写入链路会写入对应审计日志

## 4. 实现方式

本轮继续复用当前最小测试方案：

- `node:test`
- `tests/helpers/load-ts-module.mjs`
- 手写 stateful mock DB / KV / 请求对象

约束：

- 不引入新的测试框架
- 不依赖真实 D1 / KV / Wrangler 本地服务
- 优先补“写入后再读取”的长链路，不只停留在单次写接口断言
