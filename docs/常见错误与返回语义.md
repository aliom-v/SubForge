# SubForge 常见错误与返回语义

## 1. 文档目的

本文档用于把当前仓库里最常见的错误码、状态码和“容易被误解的成功语义”收成一份统一说明。

如果你想先看接口列表和请求字段，看：

- `docs/API参考与接口约定.md`

如果你想先看节点字段定义，看：

- `docs/节点字段字典.md`

如果你在排查“导入成功但托管订阅没变化”，直接看：

- `docs/节点管理与订阅使用说明.md`

## 2. 统一返回语义

### 2.1 JSON 接口

大多数管理端接口返回：

成功：

```json
{
  "ok": true,
  "data": {}
}
```

失败：

```json
{
  "ok": false,
  "error": {
    "code": "UNAUTHORIZED",
    "message": "missing bearer token"
  }
}
```

### 2.2 错误对象形状

当前统一错误结构就是 `AppErrorShape`：

- `code`
- `message`
- `details` 可选

### 2.3 `/s/:token/:target` 成功时返回纯文本

这一点最容易被误解。

公开订阅路由成功时不是 JSON，而是订阅文本；只有失败时才会返回 JSON 错误包络。

### 2.4 看到这个错误先去看哪份文档

| 你遇到的问题 | 优先去看 | 原因 |
| --- | --- | --- |
| 首次部署后卡在 setup / 登录前就报错 | `docs/部署指南.md` | 这类问题大多是 migration、绑定或部署入口不对 |
| 不确定某个接口该怎么请求、成功返回长什么样 | `docs/API参考与接口约定.md` | 这里统一写了路由、包络和示例 |
| `ss` / `hysteria2` 字段校验失败 | `docs/节点字段字典.md` | 这里直接写了字段该落到 `credentials` 还是 `params` |
| “导入成功但托管订阅没变化” | `docs/节点管理与订阅使用说明.md` | 这里已经收进当前单用户模式的主路径与高频排查项 |
| 发布前想做一次最小验收 | `docs/发布前检查清单.md` | 这里按环境、绑定、预览、公开订阅和缓存顺序检查 |

## 3. 状态码口径

### 3.1 `400`

主要表示：

- 输入校验失败
- 目标类型不支持
- 订阅编译阶段的业务错误

高频场景：

- `VALIDATION_FAILED`
- `USER_DISABLED`
- `USER_EXPIRED`
- `NO_NODES_AVAILABLE`
- `UNSUPPORTED_TARGET`

### 3.2 `401`

主要表示：

- 没带 Bearer token
- token 非法
- token 已被服务端撤销

高频消息：

- `missing bearer token`
- `invalid admin session token`
- `admin session has been revoked`

### 3.3 `403`

主要表示：

- 管理员账号不可用
- 首次安装已经完成，不允许再次 bootstrap

高频消息：

- `admin account is unavailable`
- `setup has already been completed`

### 3.4 `404`

主要表示：

- 资源不存在
- 预览数据不存在
- token 或模板不存在

高频消息：

- `user not found`
- `node not found`
- `template not found`
- `rule source not found`
- `preview data not found`
- `subscription token or template not found`

## 4. 当前错误码字典

| 错误码 | 语义 | 常见触发位置 |
| --- | --- | --- |
| `NOT_FOUND` | 资源不存在 | 用户、节点、模板、规则源、预览路由 |
| `UNAUTHORIZED` | 未认证或 token 无效 | Bearer token 缺失、非法、已撤销 |
| `FORBIDDEN` | 有 token 但当前不允许 | 管理员禁用、setup 已完成 |
| `VALIDATION_FAILED` | 输入参数不符合要求 | 大多数写接口 |
| `SUBSCRIPTION_USER_NOT_FOUND` | 订阅 token 或模板缺失 | `/s/:token/:target` |
| `USER_DISABLED` | 用户被禁用 | 预览或公开订阅编译 |
| `USER_EXPIRED` | 用户已过期 | 预览或公开订阅编译 |
| `NO_NODES_AVAILABLE` | 没有可输出节点 | 预览或公开订阅编译 |
| `TEMPLATE_NOT_FOUND` | 目标模板不存在 | 编译阶段 |
| `UNSUPPORTED_TARGET` | 目标类型不支持 | 预览或公开订阅 target 非法 |
| `RENDERER_NOT_FOUND` | 找不到渲染器 | 编译阶段 |
| `TEMPLATE_TARGET_MISMATCH` | 模板目标类型不匹配 | 编译阶段 |

## 5. 高频报错对照

### 5.1 请求体类错误

- `request body must be valid JSON`
  - 请求体不是合法 JSON

- `request body must be a JSON object`
  - JSON 解析成功了，但顶层不是对象

### 5.2 鉴权类错误

- `missing bearer token`
  - 请求头里没有 `Authorization: Bearer ...`

- `invalid admin session token`
  - token 格式或签名不对

- `admin session has been revoked`
  - 已执行过 `POST /api/admin/logout`

- `admin account is unavailable`
  - 管理员被禁用，或后台账号当前不可用

### 5.3 用户类错误

- `name is required`
  - 创建用户时没传 `name`

- `status must be active or disabled`
  - 用户状态值不在允许枚举里

- `expiresAt must be a valid datetime string`
  - 时间格式非法

- `nodeIds must be an array`
  - 绑定节点接口没有传数组

- `nodeIds must reference existing nodes: ...`
  - 绑定里包含不存在的节点 ID

### 5.4 节点类错误

- `name, protocol, server and port are required`
  - 创建节点缺少核心字段

- `port must be an integer between 1 and 65535`
  - 端口非法

- `sourceType must be manual or remote`
  - 来源类型不在允许枚举里

- `remote sourceType is not supported yet`
  - 普通节点创建 / 更新接口不允许手工把单条节点写成 `remote` 来源；远程节点只允许通过远程节点源手动同步链路写入

- `sourceId is not supported for manual nodes`
  - 手动节点不接受来源 ID

- `credentials must be a JSON object or null`
- `params must be a JSON object or null`
  - metadata 结构不合法

### 5.5 协议字段类错误

- `ss 节点需要 credentials.cipher 和 credentials.password`
- `ss 节点的 params.plugin 必须是非空字符串`
- `ss 节点暂不支持 params.plugin-opts / params.pluginOpts / params.plugins 这类复杂 plugin 字段，请继续直接核对原始 JSON`

- `hysteria2 节点需要 credentials.password`
- `hysteria2 节点的 params.sni 必须是非空字符串`
- `hysteria2 节点的 params.obfs 必须是非空字符串`
- `hysteria2 节点当前仅支持 params.obfs = "salamander"`
- `hysteria2 节点的 params["obfs-password"] 必须是非空字符串`
- `hysteria2 节点提供 params["obfs-password"] 时必须同时提供 params.obfs`
- `hysteria2 节点的 params.insecure 必须是布尔值`
- `hysteria2 节点的 params.pinSHA256 必须是字符串或非空字符串数组`
- `hysteria2 节点的 params.alpn 必须是字符串或非空字符串数组`

### 5.6 模板类错误

- `targetType must be mihomo or singbox`
- `version must be a positive integer`
- `default template must be enabled`
- `template not found`

### 5.7 规则源类错误

- `name, sourceUrl and format are required`
- `format must be text, yaml or json`
- `sourceUrl must be a valid http/https URL`
- `rule source not found`

### 5.8 订阅预览与公开订阅类错误

- `unsupported subscription target`
  - 目标类型不是 `mihomo` 或 `singbox`

- `preview data not found`
  - `userId` 或模板无法组成预览输入

- `subscription token or template not found`
  - token 不存在，或目标模板不存在

- `user is disabled`
- `user has expired`
- `no nodes available`
  - 编译阶段失败，优先去查用户状态、节点绑定和默认模板

## 6. 容易误解的成功语义

### 6.1 `POST /api/node-import/preview`

成功只表示：

- 上游抓到了内容
- 解析器给出了候选节点和错误列表

它不表示：

- 节点已经创建
- 节点已经绑定
- 订阅一定会变化

### 6.2 `POST /api/users/:id/nodes`

成功表示绑定集合已被替换。

如果你预期的是“把一个新节点追加进去”，需要自己把旧节点 ID 一起传回去。

### 6.3 `POST /api/admin/logout`

成功后旧 token 会被服务端撤销。

因此“退出成功但本地还留着 token”不影响后续鉴权失败。

### 6.4 `GET /api/preview/:userId/:target`

这是后台预览接口，会返回 JSON，并带：

- `x-subforge-preview-cache: hit | miss`

### 6.5 `GET /s/:token/:target`

这是公开订阅接口，会返回订阅文本，并带：

- `x-subforge-cache: hit | miss`

如果客户端说“订阅没变化”，不要只看节点创建是否成功，还要继续排查：

1. 是否绑定到正确用户
2. 是否使用了当前 token
3. 是否命中了旧缓存
4. 是否访问了正确 target

## 7. 推荐排查顺序

### 7.1 先看状态码

- `400` 优先看输入或业务语义
- `401` 优先看 Bearer token
- `403` 优先看账号状态或 setup 状态
- `404` 优先看资源 ID、token、模板

### 7.2 再看错误消息

当前很多错误消息已经写得足够具体，通常能直接指出字段名或缺失条件。

### 7.3 最后看链路位置

如果报错落在：

- 登录前：先看 setup / admin
- 节点录入时：先看字段字典和协议字段限制
- 预览或公开订阅时：先看用户状态、绑定、模板和缓存
