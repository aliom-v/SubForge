# SubForge 第八批自动化验证执行清单

## 1. 文档目的

本文档用于把第八批测试范围固定为“剩余写接口的边界请求与能力决策”，避免继续扩展到删除链路补齐、节点源同步实现或真实 Cloudflare 集成。

本轮只做三类工作：

1. 用户写接口的边界拒绝语义
2. 节点 / 模板 / 规则源更新接口的关键校验边界
3. 对未实现能力继续保持文档写实，而不是提前扩写承诺

当前进展：`done`

- 已补 `PATCH /api/users/:id` 对非法 `status`、非法 `expiresAt` 的回归测试
- 已修复 `POST /api/users/:id/nodes` 对缺失用户和未知节点缺少显式拒绝的问题
- 已补 `POST /api/users/:id/nodes` 对去重 / 空字符串过滤的回归测试
- 已补节点 / 模板 / 规则源更新接口的非法参数与缺失资源回归测试

---

## 2. 本轮边界

### 2.1 本轮要补

- `apps/worker/src/index.ts` 中用户节点绑定、用户更新、节点更新、模板更新、规则源更新的边界请求行为
- `tests/worker-write-integration.test.mjs` 与 `tests/template-rule-cache-routes.test.mjs` 中对应请求级回归测试
- `README.md`、`docs/自动化验证与CI计划.md`、`docs/已知问题与修复计划.md`、`.omx/plans/2026-03-07-subforge-roadmap.md` 的状态同步

### 2.2 本轮不补

- 用户 / 模板 / 规则源删除能力
- 节点源同步能力
- 真实 D1 / KV / Wrangler / Cloudflare 端到端测试
- Web 页面级测试
- 全量错误码矩阵和覆盖率平台
