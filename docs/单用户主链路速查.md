# SubForge 单用户主链路速查

## 1. 这页是干什么的

这页只保留当前单用户模式下最短、最稳的操作路径。

如果你不想先读长文档，只想知道“现在到底该怎么用”，直接看这页就够了。

更详细说明再去看：

- `docs/节点管理与订阅使用说明.md`
- `docs/部署指南.md`

## 2. 唯一主流程

按下面顺序操作：

1. 打开前端主界面，先完成 `首次安装向导`
2. 登录后进入“节点”页
3. 任选一种入口导入：
   - `节点文本导入`
   - `订阅 URL 解析`
   - `导入完整配置`
4. 检查节点列表，决定哪些节点启用
5. 如果需要链式代理，再补 `上游代理`
6. 点击 `使用当前启用节点生成托管 URL`
7. 复制页面给出的：
   - `/s/:token/mihomo`
   - `/s/:token/singbox`
8. 在客户端里直接导入 URL

## 3. 三种入口有什么区别

### 3.1 节点文本导入

适合：

- 你手上已经有分享链接
- 你只想把节点先导进来再慢慢调整

结果：

- 节点会写入列表
- 不会自动替你刷新托管输出

### 3.2 订阅 URL 解析

适合：

- 你想先看某个上游订阅到底能解析出什么节点
- 你还不确定要不要长期跟着这个上游更新

结果：

- 会先做一次抓取和解析
- 你可以先导入节点
- 如果确认要长期跟随，再点 `保存为自动同步源`

### 3.3 导入完整配置

适合：

- 你手里是整份 Mihomo / Clash YAML 或 sing-box JSON
- 你希望尽量复用其中的节点信息

结果：

- 系统会尽量提取节点
- 但当前不是完整配置编辑器

## 4. 什么时候要点“保存为自动同步源”

只在这种场景才需要：

- 你希望系统以后继续按 Cron 自动拉取同一个上游订阅

如果你只是一次性导入，通常不用保存成自动同步源。

一句话区分：

- `导入` = 只处理这一次
- `保存为自动同步源` = 以后继续定时跟这个上游

## 5. 什么时候 URL 会变

### 5.1 只更新内容，不换地址

这些场景通常只会更新链接背后的内容：

- 新增节点
- 删除节点
- 调整启用状态
- 修改节点字段
- 修改链式代理
- 自动同步源重新同步

这时通常只要再点一次：

- `使用当前启用节点生成托管 URL`

### 5.2 让旧链接失效

如果你怀疑订阅地址泄漏，或者想强制客户端换新地址，就用：

- `重置当前托管链接`

这会让旧 `/s/:token/...` 失效，并生成新地址。

## 6. 自动同步源最常看哪几个字段

先看这 4 个：

- `last_sync_status`
- `last_sync_message`
- `failure_count`
- `last_sync_details_json`

如果只想看字段解释，直接跳：

- `docs/自动同步源状态与错误说明.md`

## 7. 常见误区

- 只导入节点，不等于已经刷新托管订阅
- `订阅 URL 解析` 和 `保存为自动同步源` 不是同一件事
- 当前不是多用户机场后台，也不是完整图形化配置编辑器
- 当前公开输出仍只有 `mihomo` 和 `singbox`

## 8. 最短结论

记住一句话就够了：

先导入，后调整，再统一生成；只有需要长期跟着上游更新时，才保存为自动同步源。
