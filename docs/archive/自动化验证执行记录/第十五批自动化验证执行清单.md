# SubForge 第十五批自动化验证执行清单

## 1. 文档目的

本文档用于把第十五批工作范围固定为“节点协议向导补强”，优先解决当前节点页虽然已支持 metadata JSON，但常见协议仍需要管理员手写字段、容易填错且不易确认的问题。

本轮只做三类工作：

1. 为 Web 节点页补 `vless` / `trojan` / `vmess` 的结构化协议字段向导
2. 抽出节点 metadata helper，并补纯逻辑测试
3. 同步 README、节点使用说明、自动化验证与路线文档

当前进展：`done`

- 已补 Web 节点协议向导，并自动回填原始 JSON
- 已抽出节点 metadata helper，覆盖协议识别、metadata 解析与回填逻辑
- 已同步用户文档与状态文档

## 3. 具体执行项

### 3.1 协议向导

至少补齐：

- `vless` 的 UUID / TLS / Network / Server Name / Path
- `trojan` 的 Password / TLS / SNI
- `vmess` 的 UUID / Alter ID / TLS / Network / Server Name / Path
- 结构化字段修改后自动回填 `credentials` / `params` JSON

### 3.2 纯逻辑测试

至少覆盖：

- 协议识别
- JSON metadata 解析
- 已有 metadata 到结构化字段的回显
- 结构化字段到 metadata 对象的回填

### 3.3 文档同步

至少写清：

- 当前节点页已支持常见协议向导
- 协议向导与原始 JSON 的同步关系
- 当前仍不支持订阅链接导入与远程节点源同步

---

## 4. 完成标准

满足以下条件即可认为本轮完成：

- Web 节点页已支持常见协议向导
- `npm run typecheck`、`npm run build`、`npm run test:smoke`、`npm test` 全部通过
- 文档已同步到新的真实使用方式
