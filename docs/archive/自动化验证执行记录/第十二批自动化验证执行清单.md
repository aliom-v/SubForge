# SubForge 第十二批自动化验证执行清单

## 1. 文档目的

本文档用于把第十二批测试范围固定为“默认模板只能指向启用模板”的语义收紧，避免继续扩展到节点源同步实现、真实 Cloudflare 集成或更大的模板能力改造。

本轮只做三类工作：

1. 收紧模板创建 / 更新 / `set-default` 三条入口的默认模板语义
2. 阻止 disabled 模板被写成默认模板但接口仍返回成功的状态错觉
3. 同步自动化验证、状态与路线文档，继续保持“默认模板必须可生效”的写实描述

当前进展：`done`

- 已补 `POST /api/templates` 对 `isDefault=true` 且 `enabled=false` 的请求级回归
- 已补 `PATCH /api/templates/:id` 对 disabled 模板标记默认模板的请求级回归
- 已补 `POST /api/templates/:id/set-default` 对 disabled 模板返回 `400` 的请求级回归
- 已把模板默认模板语义收敛为“只有 enabled 模板才能成为默认模板”
