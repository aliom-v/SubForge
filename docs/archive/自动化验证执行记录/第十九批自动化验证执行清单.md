# SubForge 第十九批自动化验证执行清单

## 1. 文档目的

本文档用于把第十九批工作范围固定为“`ss://` 分享链接导入补强”，优先解决当前节点页和订阅 URL 远程预览已经支持 `vless://` / `trojan://` / `vmess://`，但对常见的 Shadowsocks 分享链接仍然会直接报不支持的问题。

本轮只做三类工作：

1. 为共享节点导入解析器新增 `ss://` 支持
2. 为 Worker 远程预览与 Web 提示补 `ss://` 文案和回归
3. 同步 README、节点使用说明、自动化验证与路线文档

当前进展：`done`

## 3. 具体执行项

### 3.1 解析器补强

至少补齐：

- 支持解析 `ss://` 标准分享链接
- 支持解析常见的 Base64 userinfo 形式
- 支持解析整段 Base64 包裹的旧式 `ss://` authority 形式
- 保持导入结果继续映射到统一字段：
  - `name`
  - `protocol`
  - `server`
  - `port`
  - `credentials`
  - `params`

### 3.2 Web / Worker 提示

至少补齐：

- 节点页支持把 `ss://` 写进导入说明和空状态提示
- 远程订阅预览支持把 `ss://` 行识别为可导入节点
- 明确当前 `ss` 只是导入支持，结构化向导仍未覆盖

### 3.3 自动化验证

至少补齐：

- `ss://` 解析纯逻辑测试
- 远程预览包含 `ss://` 行的请求级回归
- smoke 文档断言同步到第十九批

## 4. 完成标准

完成后应满足：

- 节点页与远程预览已支持 `ss://` 导入
- 当前“协议都支持了吗”的答案仍然是“还没有”，但常见导入协议已扩到 `vless` / `trojan` / `vmess` / `ss`
- `npm run typecheck`
- `npm run build`
- `npm run test:smoke`
- `npm test`
