# SubForge 第七批自动化验证执行清单

## 1. 文档目的

本文档用于把第七批测试范围固定为“更贴近本地部署前的 wrangler / 初始化回归，以及网关边界场景”，避免继续扩展到真实 Cloudflare 环境或全量路由矩阵。

本轮只做三类验证：

1. 首次安装向导的校验边界
2. Worker 网关与退出语义边界
3. 本地部署前 smoke 配置校验

当前进展：`done`

- 已补 setup/bootstrap 对非法 JSON、短用户名、短密码的边界测试
- 已补 admin/logout、OPTIONS/preflight、非 API `POST` 不回退静态资源的边界测试
- 已补 wrangler、本地 migration、`dev:worker` / `dev:web` / `test:smoke` 的 smoke 配置断言
- 已修复 Worker 顶层 `fetch` 对异步 handler 错误未统一收口的问题
- 当前仓库中的 logout 语义已在后续迭代升级为服务端撤销，本轮边界测试继续作为基础回归保留

---

## 2. 本轮边界

### 2.1 本轮要补

- `apps/worker/src/index.ts` 中 setup/bootstrap、admin/logout、OPTIONS/preflight、非 API 非 GET/HEAD 路由的边界行为
- `scripts/smoke-check.mjs`、`wrangler.toml`、`apps/worker/package.json`、`README.md`、`docs/部署指南.md` 中与本地部署前最相关的配置断言

### 2.2 本轮不补

- 真实 Wrangler 本地服务或远端发布验证
- 真实 D1 / KV 资源联调
- Web 页面级测试
- 全量错误码矩阵覆盖
- 覆盖率门槛与报告平台

---

## 3. 具体执行项

### 3.1 初始化边界

目标文件：

- `apps/worker/src/index.ts`

本轮至少覆盖：

- `POST /api/setup/bootstrap` 对非法 JSON 返回 `400`
- `POST /api/setup/bootstrap` 对过短用户名或密码返回 `400`
- 已初始化状态下继续 bootstrap 的拦截语义保持不变

### 3.2 网关与退出边界

目标文件：

- `apps/worker/src/index.ts`
- `apps/worker/src/http.ts`

本轮至少覆盖：

- `POST /api/admin/logout` 的退出路径保持可预测；当前仓库现语义为服务端撤销，旧 token 会在后续受保护请求中返回 `401`
- `OPTIONS` 请求返回 `204` 和 CORS 头
- 非 API 的 `POST` 等写请求不会错误回退到静态资源，而是返回 `404`

### 3.3 本地部署前 smoke

目标文件：

- `scripts/smoke-check.mjs`
- `wrangler.toml`
- `apps/worker/package.json`
- `README.md`
- `docs/部署指南.md`

本轮至少覆盖：

- `wrangler.toml` 中 Worker 入口、D1/KV 绑定、Cron 配置仍存在
- `apps/worker/package.json` 中 `dev` / `build` / `deploy` / `db:migrations:apply` / `db:migrations:apply:local` 脚本保持可发现
- `README.md` 与 `docs/部署指南.md` 中本地 migration、`dev:worker`、`dev:web`、`test:smoke` 路径保持写实

## 4. 实现方式

本轮继续复用当前最小测试方案：

- `node:test`
- `tests/helpers/load-ts-module.mjs`
- 手写 mock DB / Assets / 请求对象

约束：

- 不引入新的测试框架
- 不依赖真实 Wrangler 服务或远端 Cloudflare 环境
- 优先补部署前最容易出问题的边界语义，而不是继续扩展主功能面
