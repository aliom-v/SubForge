# SubForge 第二十批自动化验证执行清单

## 1. 文档目的

本文档用于把第二十批工作范围固定为“协议支持矩阵与落地路线文档补强”，优先解决当前协议支持信息虽然已经分散同步到 README、节点使用说明、实施方案与路线图，但入口分散、层级混杂，用户很难快速判断“当前到底支持到哪一层、后面准备按什么顺序落地”的问题。

本轮只做三类工作：

1. 新增统一的协议支持矩阵与落地计划文档
2. 为 README、节点使用说明、实施方案、已知问题与路线图补统一入口与一致表述
3. 为 smoke 文档检查补新文档断言

当前进展：`done`

---

## 2. 本轮边界

### 2.1 本轮要补

- `docs/协议支持矩阵与落地计划.md`
- `README.md`
- `docs/节点管理与订阅使用说明.md`
- `docs/部署指南.md`
- `docs/实施方案.md`
- `docs/已知问题与修复计划.md`
- `docs/自动化验证与CI计划.md`
- `.omx/plans/2026-03-07-subforge-roadmap.md`
- `scripts/smoke-check.mjs`

### 2.2 本轮不补

- 新协议实现
- 新协议结构化向导
- Clash / sing-box 全量配置导入
- 远程节点源同步
- UI 改版

---

## 3. 具体执行项

### 3.1 协议支持矩阵

至少补齐：

- 解释“手动录入 / 分享链接导入 / 远程预览 / 结构化向导 / 订阅输出”五个层级
- 明确当前已落地协议与未落地协议
- 明确“可渲染输出”不等于“已有完整协议产品化支持”

### 3.2 落地路线

至少补齐：

- 建议的协议落地优先级
- 每一阶段的范围与完成标准
- 当前文档承诺与未来计划的边界

### 3.3 文档入口统一

至少补齐：

- README 指向协议支持矩阵
- 节点使用说明指向协议支持矩阵
- 实施方案、已知问题与路线图引用同一套结论

---

## 4. 完成标准

完成后应满足：

- 仓库里存在可单独阅读的“协议支持矩阵与落地计划”主文档
- README、节点使用说明、实施方案、已知问题、路线图不再各写各的
- `npm run test:smoke` 通过
