# SubForge 第五批自动化验证执行清单

## 1. 文档目的

本文档用于把第五批测试范围固定为“部署前基础回归场景与更长链路的 Worker 请求级集成测试”，避免继续扩展到真实 Cloudflare 环境或全量端到端覆盖。

本轮只做三类验证：

1. 首次安装与管理员登录闭环
2. 预览与公开订阅的 cache-miss 编译链路
3. health / 静态资源回退 / Cron 触发的基础回归

当前进展：`done`

- 已补 setup/status、setup/bootstrap、admin/login、admin/me 的首次安装与登录闭环测试
- 已补 preview 与 public subscription 的 cache-miss -> cache-hit 请求级链路测试
- 已补 health、静态资源回退与 scheduled 基础回归测试
- 已同步 smoke 检查包含第五批执行清单

---

## 2. 本轮边界

### 2.1 本轮要补

- `apps/worker/src/index.ts` 中 setup/bootstrap、admin/login、admin/me、preview、public subscription、health、assets fallback、scheduled 的请求级行为
- `apps/worker/src/repository.ts` 中支撑上述链路的最小 mock 数据访问语义

### 2.2 本轮不补

- 真实 D1 / KV / Cron 集成测试
- 真实 Cloudflare 部署验证
- Web 组件级测试
- 全量 Worker 路由矩阵测试
- 覆盖率门槛与测试报告平台

---

## 3. 具体执行项

### 3.1 首次安装与登录闭环

目标文件：

- `apps/worker/src/index.ts`

本轮至少覆盖：

- `GET /api/setup/status` 在未初始化和已初始化两种状态下返回正确结果
- `POST /api/setup/bootstrap` 在空仓状态下成功创建管理员并返回 token
- `POST /api/setup/bootstrap` 在已初始化后返回 `403`
- `POST /api/admin/login` 与 `GET /api/admin/me` 能跑通首次安装后的最小登录闭环

### 3.2 预览与公开订阅 miss 路径

目标文件：

- `apps/worker/src/index.ts`
- `apps/worker/src/repository.ts`

本轮至少覆盖：

- `GET /api/preview/:userId/:target` 在 cache miss 时会编译内容、写入预览缓存并返回 `miss`
- 同一预览请求再次访问时返回 `hit`
- `GET /s/:token/:target` 在 cache miss 时会编译内容、写入订阅缓存并返回 `miss`
- 同一公开订阅请求再次访问时返回 `hit`

### 3.3 部署前基础回归

目标文件：

- `apps/worker/src/index.ts`
- `scripts/smoke-check.mjs`
- `README.md`

本轮至少覆盖：

- `GET /health` 返回 JSON 健康信息
- 非 API 的 `GET` / `HEAD` 请求会回退到 `env.ASSETS.fetch`
- `scheduled` 会调用规则源同步入口，并把 Promise 交给 `ctx.waitUntil`
- smoke 文档检查同步包含第五批执行清单

## 4. 实现方式

本轮继续复用当前最小测试方案：

- `node:test`
- `tests/helpers/load-ts-module.mjs`
- 手写 stateful mock DB / KV / fetch / ExecutionContext

约束：

- 不引入新的测试框架
- 不依赖真实 D1 / KV / 远程网络 / Cloudflare Runtime
- 只补部署前最关键的请求级链路，不追求一次性完成全量端到端
