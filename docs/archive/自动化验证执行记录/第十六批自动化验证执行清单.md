# SubForge 第十六批自动化验证执行清单

## 1. 文档目的

本文档用于把第十六批工作范围固定为“分享链接导入补强”，优先解决当前节点页虽然已经支持协议向导，但用户把常见 `vless://`、`trojan://`、`vmess://` 分享链接贴进后台时仍然没有直接导入路径的问题。

本轮只做三类工作：

1. 为 Web 节点页补分享链接解析、预览和批量创建
2. 为链接解析补纯逻辑测试
3. 同步 README、节点使用说明、自动化验证与路线文档

当前进展：`done`

## 3. 具体执行项

### 3.1 分享链接导入

至少补齐：

- 支持粘贴 `vless://` 分享链接
- 支持粘贴 `trojan://` 分享链接
- 支持粘贴 `vmess://` Base64 分享链接
- 支持多行粘贴并逐行解析
- 支持在导入前预览节点基础信息
- 支持把解析结果载入创建表单或直接批量创建

### 3.2 数据映射

至少保证：

- 导入结果映射到当前后端真实接受的数据形状：
  - `name`
  - `protocol`
  - `server`
  - `port`
  - `credentials`
  - `params`
- 常见字段如 `uuid` / `password` / `tls` / `network` / `servername` / `path` / `sni` 与当前协议向导保持一致

### 3.3 文档同步

至少写清：

- 当前已支持的导入类型是“单条或多条分享链接粘贴导入”
- 当前仍不支持“订阅 URL 远程抓取”“远程节点源同步”
- 导入完成后仍需要到“用户”页完成节点绑定

---

## 4. 完成标准

满足以下条件即可认为本轮完成：

- 节点页已支持常见分享链接导入
- `npm run typecheck`、`npm run build`、`npm run test:smoke`、`npm test` 全部通过
- 文档已更新为新的真实使用边界
