## 1. 命名

- `Sub`：订阅
- `Forge`：锻造、生成
- 这个名字表达的是“把节点整理后生成可直接使用的订阅”

## 2. 当前方向

SubForge 当前只服务一名管理员 / 一套个人托管订阅。

主线始终只有这一条：

1. 部署到 Cloudflare
2. 导入节点文本、订阅 URL 或完整配置
3. 调整节点启用状态和链式代理关系
4. 统一生成 `mihomo` / `singbox` 托管 URL
5. 让客户端直接拉取使用

## 4. 仍保留的长期设想

- 保持 Cloudflare Worker + D1 + KV + Cron 的无常驻服务器形态
- 持续提升导入稳定性、托管输出稳定性和自动同步体验
- 继续把内部实现细节收在受保护接口后面，不让它们干扰单人使用主路径
