# SubForge 第十七批自动化验证执行清单

## 1. 文档目的

本文档用于把第十七批工作范围固定为“订阅 URL 远程抓取预览导入”，补上当前节点页虽然已经支持手动录入与分享链接粘贴导入，但用户手里如果只有远程订阅链接，仍然需要先在外部工具中手动展开后再粘贴的问题。

本轮只做三类工作：

1. 为 Worker 增加管理端订阅 URL 远程抓取预览接口
2. 为 Web 节点页补远程订阅预览与批量创建入口
3. 同步 README、节点使用说明、自动化验证与路线文档

当前进展：`done`

## 3. 具体执行项

### 3.1 共享导入解析器

至少补齐：

- 把 `vless://` / `trojan://` / `vmess://` 的解析逻辑抽到 `packages/core`
- Web 与 Worker 复用同一份 `parseNodeImportText`
- 继续把节点规范化到统一结构：
  - `name`
  - `protocol`
  - `server`
  - `port`
  - `credentials`
  - `params`

### 3.2 Worker 远程预览接口

至少补齐：

- 新增受管理员鉴权保护的预览接口：`POST /api/node-import/preview`
- 请求体至少接受：
  - `sourceUrl`
- 仅允许 `http://` / `https://`
- Worker 远程抓取文本后直接做解析预览
- 返回：
  - `sourceUrl`
  - `upstreamStatus`
  - `durationMs`
  - `fetchedBytes`
  - `lineCount`
  - `nodes`
  - `errors`
- 仅做预览与审计，不写入节点、不持久化远程源、不失效缓存

### 3.3 Web 节点页交互

至少补齐：

- 新增“导入订阅 URL”面板
- 允许输入远程订阅链接并请求 Worker 预览
- 展示远程抓取摘要、解析错误与可导入节点列表
- 允许把预览出的单条节点载入创建表单
- 允许批量创建解析成功的节点
- 明确提示：
  - 这不是远程节点源同步
  - 创建后仍需手动绑定用户

### 3.4 自动化验证

至少补齐：

- 分享链接解析测试继续可通过
- Worker 新增远程预览接口回归：
  - 成功抓取并解析
  - 非法 URL 拒绝
  - 上游返回异常时拒绝
  - 成功预览会写审计
- smoke 文档断言同步到第十七批

## 4. 完成标准

完成后应满足：

- 节点页已同时支持：
  - 手动录入
  - 分享链接粘贴导入
  - 订阅 URL 远程抓取预览导入
- 远程导入仅是一次性预览与创建，不会伪装成“已实现远程同步”
- Worker 与 Web 共享同一份解析逻辑，避免协议映射漂移
- `npm run typecheck`
- `npm run build`
- `npm run test:smoke`
- `npm test`

---

## 5. 风险提示

本轮仍保留以下现实限制：

- 仅支持把远程文本中的 `vless://` / `trojan://` / `vmess://` 行解析成标准节点
- 如果上游订阅内容为其他协议或完整配置文件，当前不会自动完整转换
- 远程 URL 预览成功不代表节点一定可连通，仍需要人工核对 `credentials` / `params`
