# SubForge 文档导航与阅读顺序

## 1. 这份文档是做什么的

如果你第一次接手这个项目，或者刚部署完但不知道下一步该看哪份文档，先看这里。

这份文档只做两件事：

1. 告诉你先读哪几份文档
2. 告诉你不同场景应该跳去哪里

---

## 2. 第一次接手这个项目应该看哪里

推荐顺序：

1. `README.md`
2. `docs/文档导航与阅读顺序.md`
3. `docs/部署指南.md`
4. `docs/节点管理与订阅使用说明.md`
5. `docs/协议支持矩阵与落地计划.md`
6. `docs/已知问题与修复计划.md`

原因：

- `README.md` 先给你当前仓库的总体能力与最小使用路径
- 本文档负责把剩余文档分工和跳转关系讲清楚
- `docs/部署指南.md` 负责环境启动、Cloudflare 部署与首次初始化
- `docs/节点管理与订阅使用说明.md` 负责后台实际操作流程
- `docs/协议支持矩阵与落地计划.md` 负责解释协议支持到哪一层
- `docs/已知问题与修复计划.md` 负责解释哪些是当前限制，哪些是后续顺序

## 4. 按角色阅读路径

### 4.1 开发者

推荐顺序：

1. `README.md`
2. `docs/API参考与接口约定.md`
3. `docs/节点字段字典.md`
4. `docs/协议支持矩阵与落地计划.md`
5. `docs/自动化验证与CI计划.md`
6. `docs/第二十二批自动化验证执行清单.md`
7. `docs/第二十六批自动化验证执行清单.md`

适用场景：

- 想先知道当前接口长什么样
- 想确认节点字段应该落到哪里
- 想判断补协议时要改哪些实现和测试点

### 4.2 部署 / 运维

推荐顺序：

1. `README.md`
2. `docs/部署指南.md`
3. `docs/发布前检查清单.md`
4. `docs/常见错误与返回语义.md`
5. `docs/导入成功但订阅没变化排障指南.md`
6. `docs/已知问题与修复计划.md`

适用场景：

- 想先把环境跑起来
- 想快速判断当前错误是 migration、鉴权、模板还是订阅链路问题
- 想知道当前限制和下一步风险点

### 4.3 运营 / 后台使用

推荐顺序：

1. `README.md`
2. `docs/节点管理与订阅使用说明.md`
3. `docs/节点协议示例库.md`
4. `docs/节点字段字典.md`
5. `docs/导入成功但订阅没变化排障指南.md`

适用场景：

- 想按后台页签实际操作一遍
- 想直接复制协议样例
- 想在录入节点时知道字段该放进 `credentials` 还是 `params`

## 5. 核心文档分工

- `README.md`：仓库总览、能力摘要、最小使用路径
- `docs/部署指南.md`：环境、部署、migration、首次初始化、部署后常见问题
- `docs/节点管理与订阅使用说明.md`：后台页签操作顺序、节点导入、绑定、预览
- `docs/协议支持矩阵与落地计划.md`：协议支持层级、当前边界、下一阶段顺序
- `docs/API参考与接口约定.md`：管理端 API、公开订阅、返回包络和关键请求体
- `docs/节点字段字典.md`：节点字段、metadata 与协议映射
- `docs/常见错误与返回语义.md`：错误码、状态码和高频报错对照
- `docs/发布前检查清单.md`：部署完成后的最小验收与缓存检查顺序
- `docs/节点协议示例库.md`：可复制样例和常见错误语义对照
- `docs/导入成功但订阅没变化排障指南.md`：导入成功后的链路排查
- `docs/已知问题与修复计划.md`：真实限制、风险和后续优先级
- `docs/自动化验证与CI计划.md`：测试覆盖、CI、批次回归
- `项目介绍.md`：产品设想与长期方向，不等同于当前已实现状态

---

1. `README.md`
