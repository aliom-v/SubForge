# SubForge 第十一批自动化验证执行清单

## 1. 文档目的

本文档用于把第十一批测试范围固定为“节点写接口的未实现字段语义收紧”，避免继续扩展到节点源同步实现、真实 Cloudflare 集成或更大的后台交互改造。

本轮只做三类工作：

1. 收紧 `POST /api/nodes` 与 `PATCH /api/nodes/:id` 的 `sourceType` / `sourceId` 语义
2. 拒绝非对象的 `credentials` / `params` 写入，并允许更新时显式清空
3. 同步自动化验证、状态与路线文档，继续保持“当前仅支持手动节点”的写实描述

当前进展：`done`

- 已补 `POST /api/nodes` 对 `sourceType=remote`、手动节点携带 `sourceId`、非对象 `credentials` 的请求级回归
- 已补 `PATCH /api/nodes/:id` 对 `sourceType=remote`、非对象 `params` 的请求级回归
- 已补 `PATCH /api/nodes/:id` 传 `credentials: null` / `params: null` 时显式清空元数据的回归
- 已把节点写接口收敛为“仅接受手动节点录入；节点源同步继续保留为后续能力决策”

## 2. 本轮边界

### 2.1 本轮要补

- `apps/worker/src/index.ts` 中节点创建 / 更新接口的 `sourceType`、`sourceId`、`credentials`、`params` 校验语义
- `apps/worker/src/repository.ts` 中节点 metadata 的 `null` 持久化与清空语义
- `tests/worker-write-integration.test.mjs` 中对应请求级回归
- `README.md`、`docs/实施方案.md`、`docs/自动化验证与CI计划.md`、`docs/已知问题与修复计划.md`、`.omx/plans/2026-03-07-subforge-roadmap.md` 的状态同步

### 2.2 本轮不补

- 节点源同步实现
- 新的节点源数据模型或额外 migration
- Web 页面级编辑能力扩展
- 真实 Wrangler / D1 / Cloudflare 端到端验证
- 全量错误码矩阵和覆盖率平台
