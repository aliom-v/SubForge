## 2. 先记住这条主线

当前单用户主界面基本围绕“节点”页使用，不需要单独的用户管理页。

一句话记住：

- 先导入
- 再调整节点和链式代理
- 最后统一生成托管 URL
- 只有需要长期跟着上游更新时，才保存为自动同步源

### 2.1 几个关键动作分别会做什么

| 动作 | 会写入节点列表 | 会保存长期同步关系 | 会立刻刷新托管 URL |
| --- | --- | --- | --- |
| `导入节点 N` | 会 | 不会 | 不会 |
| `抓取并导入节点` | 会 | 不会 | 不会 |
| `导入配置` | 会 | 不会 | 不会 |
| `保存为自动同步源` | 会，按同步结果回写 | 会 | 不会 |
| `使用当前启用节点生成托管 URL` | 不新增节点 | 不涉及 | 会 |
| `重置当前托管链接` | 不涉及 | 不涉及 | 会，且地址会变 |

最容易混淆的一点：

- `导入` 只是在改节点列表
- `使用当前启用节点生成托管 URL` 才是在刷新客户端真正拉取的订阅内容

## 3. 三种导入入口怎么选

| 你的场景 | 用哪个入口 | 结果 |
| --- | --- | --- |
| 手上已经有分享链接、Base64 订阅文本或 JSON 节点清单 | `节点文本导入` | 解析后写入节点列表，方便后续再调整 |
| 手上只有一个上游订阅 URL，想先看看能解析出什么 | `订阅 URL 解析` | 先抓取并预览，可选一次性导入，或继续保存为自动同步源 |
| 手里是整份 Mihomo / Clash YAML 或 sing-box JSON | `导入完整配置` | 尽量提取节点，并复用其中可收敛的输出骨架 |

补充说明：

- `订阅 URL 解析 -> 抓取并导入节点` 是一次性快照
- `订阅 URL 解析 -> 保存为自动同步源` 才会建立长期定时同步关系
- `导入完整配置` 不是完整配置编辑器，只是把能托管的部分收进当前主链路

## 4. 托管 URL 什么时候会变

### 4.1 多数情况只会更新内容，不会换地址

下面这些场景通常只要重新执行一次 `使用当前启用节点生成托管 URL`：

- 新增或删除节点
- 调整启用状态
- 修改节点名字或协议字段
- 修改链式代理
- 自动同步源同步出了新节点

这时通常更新的是链接背后的内容，不是 `/s/:token/...` 这条地址本身。

### 4.2 只有重置链接才会让旧地址失效

如果你怀疑托管链接泄漏，或者想强制客户端换新地址，就用：

- `重置当前托管链接`

这一步会轮换 token，旧 `/s/:token/mihomo` 和 `/s/:token/singbox` 会失效。

## 5. 节点编辑和链式代理

当前节点页承担的是“节点编辑器”，不是“完整配置编辑器”。

你可以在这里长期维护的主要是：

- 节点名称
- 是否启用
- 协议字段
- 上游代理关系

链式代理当前统一收敛成：

```json
{
  "upstreamProxy": "上游节点或组名"
}
```

对应关系：

- Mihomo / Clash 的 `dialer-proxy`
- sing-box 的 `detour`

页面里的链式代理拓扑会继续帮助你检查：

- 是否引用了不存在的上游
- 是否形成循环链路
- 当前链路最终会经过哪些上游节点

## 6. 页面里的几个状态词该怎么理解

- `当前启用节点`
  - 现在节点列表里被标记为启用的集合
- `当前已托管绑定`
  - 上一次成功生成托管 URL 时真正写入托管订阅的节点集合
- `需要重新生成`
  - 说明你已经改了节点列表，但客户端当前拿到的仍是上一版托管内容
- `自动同步源`
  - 存在数据库里的长期上游订阅来源，会立即同步一次，并继续按 Cron 定时拉取

如果你想看这些词的更完整定义和彼此关系，直接跳：

- `docs/单用户术语速查.md`

## 7. 当前限制

当前边界仍然很明确：

- 当前公开输出只有 `mihomo` 和 `singbox`
- 当前不是完整的 Clash / Mihomo / sing-box 图形化配置编辑器
- `抓取并导入节点` 仍是一轮即时导入，不会自动转成长期同步任务
- 内部用户、模板、规则骨架、缓存和审计对象仍存在，但默认都由系统自动维护，不要求手动操作
- 当前主目标仍是“自己部署、自己用”的单用户托管链路，不是多用户机场管理平台

## 8. 出问题先看哪里

- 导入成功但页面没有托管 URL：
  - 先确认有没有执行 `使用当前启用节点生成托管 URL`
- 自动同步源显示失败、跳过或状态看不懂：
  - 看 `docs/自动同步源状态与错误说明.md`
- 字段不确定该填哪里：
  - 看 `docs/节点字段字典.md`
- 想找可复制的协议样例：
  - 看 `docs/节点协议示例库.md`
- 想排查缓存、鉴权、限流或 HTML 回退：
  - 看 `docs/API错误码与响应头说明.md`
- 想看更完整的症状驱动排障：
  - 看 `docs/排障与常见问题.md`

## 9. 一句话结论

把这页压缩成一句话就是：

导入只会改节点，统一生成才会改订阅；长期跟上游更新时，才保存为自动同步源。

不要把当前版本当成完整多用户管理平台，也不要把 `订阅 URL -> 抓取并导入节点` 和 `保存为自动同步源` 误解成同一件事。
