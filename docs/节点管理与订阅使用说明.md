# SubForge 节点管理与订阅使用说明

## 1. 文档目的

本文档只解释当前单用户托管模式下最真实的使用路径：

1. 把节点或配置导进来
2. 让系统自动生成托管 URL
3. 让客户端直接通过 URL 使用

如果你想先看当前产品目标与非目标，看：

- `docs/单用户托管模式需求.md`

如果你想先查字段该放哪里，看：

- `docs/节点字段字典.md`

如果你想看接口和错误语义，看：

- `docs/API参考与接口约定.md`
- `docs/常见错误与返回语义.md`

---

## 2. 推荐使用路径

当前后台默认应该只围绕“节点”页使用。

推荐顺序：

1. 登录后台
2. 进入“节点”
3. 在三种入口里选择一种：
   - `节点文本导入`
   - `订阅 URL 解析`
   - `导入完整配置`
4. 点击 `导入并生成托管 URL`、`直接导入并生成托管 URL`，或在需要长期跟随上游时点击 `保存并立即同步`
5. 复制页面给出的 `/s/:token/mihomo` 或 `/s/:token/singbox`
6. 在客户端里直接填入 URL

说明：

系统内部仍会自动维护托管身份、输出骨架和缓存，但这条链路不要求你手动去管理用户、模板、规则集。

---

## 3. 三种导入入口

### 3.1 节点文本导入

适合直接粘贴：

- 分享链接
  - `vless://`
  - `trojan://`
  - `vmess://`
  - `ss://`
  - `ssr://`
  - `tuic://`
  - `hysteria2://` / `hy2://`
- Base64 包装的订阅文本
- JSON 节点清单

页面会反馈：

- 当前识别的是明文还是 Base64
- 有效行数
- 成功识别出的节点列表
- 解析失败的错误摘要

### 3.2 订阅 URL 解析

适合你手里只有一个上游订阅链接的场景。

当前行为是：

- Worker 远程抓取一次上游内容
- 当场解析并展示预览
- 允许直接导入并生成托管 URL
- 允许保存成自动拉取任务并立即同步

如果你点击 `保存并立即同步`：

- 这个 URL 会被持久化到数据库
- Worker 会立刻同步一次
- 之后按当前 `wrangler.toml` 的 Cron 配置继续定时拉取
- 已经使用这条源的托管订阅会跟着刷新到最新节点集合

### 3.3 导入完整配置

适合直接粘贴：

- 完整 Mihomo / Clash YAML
- 完整 sing-box JSON

当前行为是：

- 提取其中能识别的节点
- 尽量把关键输出结构一起带进托管链路
- 直接用于生成托管 URL

注意：

这里不是完整配置编辑器。当前目标是“导入后可托管”，不是“原始配置文件的全量可视化管理”。

---

## 4. 导入后会发生什么

当你选择“导入并生成托管 URL”时，系统会自动：

- 创建或复用同一组个人托管链接
- 将这次识别到的节点写入节点列表
- 将这些节点收进当前托管订阅
- 返回当前站点域名下的订阅地址

也就是说：

- 你如果已经把 Worker 绑到自己的域名
- 页面里生成的 URL 就会直接是你自己的域名

当前输出目标只有两种：

- `mihomo`
- `singbox`

---

## 5. 解析与诊断

当前解析链路重点覆盖下面几件事：

- 识别分享链接、本地 Base64 文本、JSON 节点清单
- 识别完整 Mihomo / Clash YAML 与完整 sing-box JSON
- 对明显错误输入给出可读错误
- 对远程抓取给出 HTTP 状态、耗时、字节数和内容类型提示

如果配置里存在链式代理关系：

- Mihomo `dialer-proxy`
- sing-box `detour`

系统会统一落到：

```json
{
  "upstreamProxy": "上游节点或组名"
}
```

节点页里的“链式代理拓扑”会继续帮助你检查：

- 节点是否连到了正确上游
- 是否存在缺失引用
- 是否出现循环链路

---

## 6. 当前限制

当前最重要的边界只有几条：

- 不支持完整的 Clash / Mihomo / sing-box 配置编辑器
- 不支持通用第三方所有订阅格式输出
- 当前公开输出仍只有 `mihomo` 和 `singbox`

如果你的需求是“把上游订阅存进来，然后系统自己定时拉取更新”，当前版本已经覆盖这一段；但它仍然不是多用户机场后台，也不是通用订阅转换平台。

---

## 7. 常见排查

### 7.1 导入成功但页面没有托管 URL

先确认你点的是不是：

- `导入并生成托管 URL`
- 或 `直接导入并生成托管 URL`

如果你点的是“仅导入节点”，系统只会更新节点列表，不会主动刷新当前托管链接。

### 7.2 订阅 URL 抓取成功，但没解析出节点

高频原因：

- 上游返回的是 HTML 登录页或报错页
- 上游内容为空
- 上游内容不是当前支持的分享链接、JSON 节点清单或完整配置

### 7.3 客户端里看起来还是旧内容

优先检查：

1. 当前使用的是否是页面最新展示的托管 URL
2. 客户端是否缓存了旧结果
3. 这次导入是否真的解析出了节点
4. 页面上的“输出检查”是否通过

如果你用的是自动拉取任务，再补看两件事：

5. 任务列表里最近一次同步状态是不是 `failed`
6. 上游 URL 现在返回的是不是登录页、空内容或新的不兼容格式

### 7.4 协议字段不对

先看：

- `docs/节点字段字典.md`
- `docs/节点协议示例库.md`

---

## 8. 一句话结论

当前最适合的使用方式就是：

- 三种入口任选一种导入
- 让系统自动生成托管 URL
- 如果是上游订阅 URL，需要长期跟着更新时就点 `保存并立即同步`
- 客户端直接拉取

不要把当前版本当成完整多用户后台，但也不要再把订阅 URL 面板误解成只有一次性预览，它现在已经支持保存和定时同步。
