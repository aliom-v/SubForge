# SubForge 节点管理与订阅使用说明

## 1. 文档目的

本文档只解释当前单用户托管模式下最真实的使用路径：

1. 把节点或配置导进来
2. 按需要调整启用节点或链式代理关系
3. 生成或刷新托管 URL
4. 让客户端直接通过 URL 使用

如果你想先看当前产品目标与非目标，看：

- `docs/单用户托管模式需求.md`

如果你想先查字段该放哪里，看：

- `docs/节点字段字典.md`

如果你只是排查使用问题，优先看：

- `docs/API错误码与响应头说明.md`
- `docs/排障与常见问题.md`

API、架构、ER、CI 一类文档都属于仓库内部参考，不属于这条单人使用主线。

---

## 2. 推荐使用路径

当前后台默认应该只围绕“节点”页使用。

推荐顺序：

1. 登录后台
2. 进入“节点”
3. 在三种入口里选择一种：
   - `节点文本导入`
   - `订阅 URL 解析`
   - `导入完整配置`
4. 先把节点导入到节点列表，或在需要长期跟随上游时点击 `保存为自动同步源`
5. 视情况调整节点启用状态、节点字段或链式代理关系
6. 点击 `使用当前启用节点生成托管 URL`
7. 复制页面给出的 `/s/:token/mihomo` 或 `/s/:token/singbox`
8. 在客户端里直接填入 URL
9. 如果需要让旧链接失效，直接在“节点”页使用 `重置当前托管链接`

说明：

系统内部仍会自动维护托管身份、输出骨架和缓存，但这条链路不要求你手动去管理用户、模板、规则集。

补充说明：

- 公开订阅不会复用管理员登录态
- 系统会单独维护一个内部托管身份来承载 `/s/:token/...`
- 如果需要轮换这条公开订阅链接，直接在“节点”页重置当前托管链接即可，不需要先进入单独的用户页

## 3. 三种导入入口

### 3.1 节点文本导入

适合直接粘贴：

- 分享链接
  - `vless://`
  - `trojan://`
  - `vmess://`
  - `ss://`
  - `ssr://`
  - `tuic://`
  - `hysteria2://` / `hy2://`
- Base64 包装的订阅文本
- JSON 节点清单

页面会反馈：

- 当前识别的是明文还是 Base64
- 有效行数
- 成功识别出的节点列表
- 解析失败的错误摘要

当前按钮语义：

- `导入节点 N`
  - 只把这次识别出的节点写入节点列表
  - 不会立即刷新当前托管订阅

### 3.2 订阅 URL 解析

适合你手里只有一个上游订阅链接的场景。

当前行为是：

- Worker 远程抓取一次上游内容
- 将上游内容按当前支持格式解析为节点
- 允许一次性抓取并导入节点
- 允许保存成自动同步源并立即同步

当前按钮语义：

- `抓取并导入节点`
  - 会先抓取、解析，再把节点作为一次性导入写入节点列表
  - 这条路径是即时快照，不会把这个 URL 保存成后续自动同步源
- `保存为自动同步源`
  - 会把这个 URL 持久化为自动同步源
  - Worker 会立刻同步一次，之后继续按当前 Cron 配置定时拉取
  - 这条来源同步出的节点会进入节点列表，后续仍应通过统一生成步骤刷新托管订阅

如果你点击 `保存为自动同步源`：

- 这个 URL 会被持久化到数据库
- Worker 会立刻同步一次
- 之后按当前 `wrangler.toml` 的 Cron 配置继续定时拉取
- 这条来源同步出的节点会继续回到节点列表，后续仍需通过统一生成步骤刷新托管订阅

### 3.3 导入完整配置

适合直接粘贴：

- 完整 Mihomo / Clash YAML
- 完整 sing-box JSON

当前行为是：

- 提取其中能识别的节点
- 尽量把关键输出结构一起带进托管链路
- 把节点写入节点列表
- 把完整配置里的关键输出结构写入当前自动托管模板骨架

当前按钮语义：

- `导入配置`
  - 会把提取出的节点写入节点列表
  - 并在目标类型匹配时把完整配置里的关键输出结构带进当前自动托管模板
  - 不会单独绕过“使用当前启用节点生成托管 URL”这一步

注意：

这里不是完整配置编辑器。当前目标是“导入后可托管”，不是“原始配置文件的全量可视化管理”。

## 4. 导入、调整和生成之间会发生什么

### 4.1 只导入节点

如果你点击的是：

- `导入节点 N`
- `抓取并导入节点`
- `导入配置`

系统会：

- 把这次识别到的节点写入节点列表
- 保留你后续手动调整节点的空间
- 不主动刷新当前托管订阅

### 4.2 按当前启用节点统一生成托管 URL

如果你点击的是：

- `使用当前启用节点生成托管 URL`

系统会：

- 复用当前托管输出骨架
- 读取节点列表里当前所有启用节点
- 用这组启用节点整体刷新当前托管订阅
- 生成或更新 `/s/:token/mihomo` 与 `/s/:token/singbox`

如果你只是：

- 改了节点
- 新增或删除了节点
- 调整了启用状态

通常只需要重新执行这一步，URL 一般保持不变，更新的是链接背后的内容。

如果你要的是：

- 让旧链接立即失效
- 强制客户端换一条新的订阅地址

那就使用“节点”页里的 `重置当前托管链接`。

这条路径最适合：

- 你已经从多个入口导入过节点
- 你刚手动调整过启用状态
- 你想按当前节点列表的真实状态重新生成订阅

### 4.3 保存为自动同步源

如果你点击的是：

- `保存为自动同步源`

系统会：

- 把上游订阅 URL 持久化为自动同步源
- 立刻同步一次
- 后续继续按当前 `wrangler.toml` 的 Cron 配置定时拉取
- 把这条来源同步出的节点写入节点列表

补充说明：

- `订阅 URL -> 抓取并导入节点` 只是一次性抓取快照，不等于创建自动同步源
- `订阅 URL -> 保存为自动同步源` 才会建立长期自动同步关系
- 无论你走哪条导入路径，最终都应回到 `使用当前启用节点生成托管 URL`
- 如果你已经把 Worker 绑到自己的域名，页面里生成的 URL 就会直接是你自己的域名
- 当前输出目标仍只有 `mihomo` 和 `singbox`

## 5. 解析与诊断

当前解析链路重点覆盖下面几件事：

- 识别分享链接、本地 Base64 文本、JSON 节点清单
- 识别完整 Mihomo / Clash YAML 与完整 sing-box JSON
- 对明显错误输入给出可读错误
- 对远程抓取给出 HTTP 状态、耗时、字节数和内容类型提示

如果配置里存在链式代理关系：

- Mihomo `dialer-proxy`
- sing-box `detour`

系统会统一落到：

```json
{
  "upstreamProxy": "上游节点或组名"
}
```

节点页里的“链式代理拓扑”会继续帮助你检查：

- 节点是否连到了正确上游
- 是否存在缺失引用
- 是否出现循环链路

## 6. 当前限制

当前最重要的边界只有几条：

- 不支持完整的 Clash / Mihomo / sing-box 配置编辑器
- 不支持通用第三方所有订阅格式输出
- 当前公开输出仍只有 `mihomo` 和 `singbox`
- `订阅 URL -> 抓取并导入节点` 当前仍是一轮即时导入，不会自动转成远程源任务
- 当前托管 URL 卡片会按后端已保存的自动托管用户、节点绑定和预览结果自动恢复；但如果你只改了节点列表、还没重新执行统一生成，卡片展示的仍是上一次已保存的托管绑定

如果你的需求是“把上游订阅存进来，然后系统自己定时拉取更新”，当前版本已经覆盖这一段；但它仍然不是多用户机场后台，也不是通用订阅转换平台。

## 7. 常见排查

### 7.1 导入成功但页面没有托管 URL

先确认你点的是不是：

- `使用当前启用节点生成托管 URL`
- 或 `保存为自动同步源` 后又重新执行了统一生成

如果你点的是：

- `导入节点 N`
- `抓取并导入节点`
- `导入配置`

系统只会更新节点列表和托管模板骨架，不会主动刷新当前托管链接。

另外，当前托管 URL 卡片现在会在页面刷新后按当前已保存托管状态自动恢复；但如果你刚改的是节点列表、还没重新执行统一生成，这张卡片展示的仍会是上一次已保存的托管绑定，这是预期行为。

后台现在也会直接提示“当前启用节点”和“当前已托管绑定”是否一致，并显示 `未生成` / `已同步` / `需要重新生成`；如果页面提示需要重新生成，就说明你已经改了节点列表，但客户端当前拿到的仍是上一版托管绑定。

节点页顶部还会把这条主流程拆成三张状态卡片，分别对应“导入”“调整”“统一生成”，方便你在刷新页面后继续从当前步骤往下走。

### 7.2 订阅 URL 抓取成功，但没解析出节点

高频原因：

- 上游返回的是 HTML 登录页或报错页
- 上游内容为空
- 上游内容不是当前支持的分享链接、JSON 节点清单或完整配置

### 7.3 客户端里看起来还是旧内容

优先检查：

1. 当前使用的是否是页面最新展示的托管 URL
2. 客户端是否缓存了旧结果
3. 这次导入是否真的解析出了节点
4. 页面上的“输出检查”是否通过

如果你用的是自动同步源，再补看两件事：

5. 任务列表里最近一次同步状态是不是 `failed`
6. 上游 URL 现在返回的是不是登录页、空内容或新的不兼容格式

### 7.4 协议字段不对

先看：

- `docs/节点字段字典.md`
- `docs/节点协议示例库.md`

---

## 8. 一句话结论

当前最适合的使用方式就是：

- 三种入口任选一种先导入
- 先把节点调整到你真正想输出的状态
- 再点 `使用当前启用节点生成托管 URL`
- 如果是上游订阅 URL，需要长期跟着更新时就点 `保存为自动同步源`
- 客户端直接拉取

不要把当前版本当成完整多用户后台，也不要把 `订阅 URL -> 抓取并导入节点` 和 `保存为自动同步源` 误解成同一件事。
