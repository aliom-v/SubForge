# SubForge 节点字段字典

## 1. 文档目的

本文档用于把当前节点模型、`credentials` / `params` 语义，以及常见协议字段映射收成一份统一字典。

如果你主要关心接口路径与请求包络，看：

- `docs/API参考与接口约定.md`

如果你主要关心报错和状态码，看：

- `docs/API错误码与响应头说明.md`

---

## 2. 节点统一结构

当前节点记录以 `NodeRecord` 为基线，核心字段如下：

| 字段 | 类型 | 含义 | 当前口径 |
| --- | --- | --- | --- |
| `id` | `string` | 节点 ID | 数据库生成 |
| `name` | `string` | 展示名称 | 创建时必填 |
| `protocol` | `string` | 协议名 | 当前常见为 `vless` / `trojan` / `vmess` / `ss` / `ssr` / `tuic` / `hysteria2` |
| `server` | `string` | 地址或域名 | 创建时必填 |
| `port` | `number` | 端口 | 必须在 `1..65535` |
| `sourceType` | `manual \| remote` | 来源类型 | 当前写接口只接受 `manual` |
| `sourceId` | `string \| null` | 来源 ID | 当前手动节点不支持 |
| `enabled` | `boolean` | 是否启用 | 默认启用 |
| `lastSyncAt` | `string \| null` | 最近同步时间 | 预留给远程来源 |
| `credentials` | `object \| null` | 核心凭据 | 协议必需 secret 通常放这里 |
| `params` | `object \| null` | 附加参数 | TLS、传输、plugin、obfs 等通常放这里 |

最重要的分工：

- `credentials` 放“缺了就基本连不上”的核心凭据
- `params` 放传输层、TLS 层、plugin 层和额外选项

---

## 3. metadata 统一规则

### 3.1 `credentials` / `params` 必须是对象或 `null`

当前写接口和前端表单都只接受：

- JSON 对象
- `null`
- 留空

不接受：

- 数组
- 纯字符串
- 其他原始类型

### 3.2 `hy2` 会归一化为 `hysteria2`

分享链接导入和协议字段校验都会把 `hy2` 统一当成 `hysteria2`。

### 3.3 结构化向导不是“另一个数据源”

当前后台里的结构化协议向导，本质上只是帮助你回填 `credentials` / `params`。

真正提交给后端的仍然是这两块 JSON。

### 3.4 手动录入、分享链接导入、远程预览导入共用同一套节点形状

无论来源是什么，最终都会落成：

- `name`
- `protocol`
- `server`
- `port`
- `credentials`
- `params`

## 4. 各协议常见字段映射

### 4.1 `vless`

常见落点：

- `credentials.uuid`
- `params.tls`
- `params.network`
- `params.servername`
- `params.path`
- `params.sni`

适合放在 `credentials` 的内容：

- `uuid`

适合放在 `params` 的内容：

- 传输层和 TLS 相关字段

### 4.2 `trojan`

常见落点：

- `credentials.password`
- `params.sni`
- `params.network`
- `params.path`

### 4.3 `vmess`

常见落点：

- `credentials.uuid`
- `params.alterId`
- `params.tls`
- `params.network`
- `params.servername`
- `params.path`

### 4.4 `ss`

当前常见字段：

- `credentials.cipher`
- `credentials.password`
- `params.plugin`

当前限制：

- `credentials.cipher` 和 `credentials.password` 都是必填
- `params.plugin` 如果存在，必须是非空字符串
- `params.plugin-opts`
- `params.pluginOpts`
- `params.plugins`

以上复杂 plugin 字段当前仍不支持，会直接返回校验错误

### 4.5 `ssr`

当前常见字段：

- `credentials.cipher`
- `credentials.password`
- `credentials.protocol`
- `credentials.obfs`
- `params["protocol-param"]`
- `params["obfs-param"]`

当前限制：

- `credentials.cipher` / `credentials.password` / `credentials.protocol` / `credentials.obfs` 都是必填
- `params["protocol-param"]` 如果存在，必须是非空字符串
- `params["obfs-param"]` 如果存在，必须是非空字符串

### 4.6 `tuic`

当前常见字段：

- `credentials.uuid`
- `credentials.password`
- `params.sni`
- `params.alpn`
- `params.insecure`
- `params["congestion-controller"]`
- `params["udp-relay-mode"]`
- `params["disable-sni"]`
- `params.heartbeat`
- `params["request-timeout"]`
- `params["reduce-rtt"]`

当前限制：

- `credentials.uuid` 和 `credentials.password` 都是必填
- `params.sni` 如果存在，必须是非空字符串
- `params.alpn` 必须是字符串或非空字符串数组
- `params.insecure` / `params["disable-sni"]` / `params["reduce-rtt"]` 如果存在，必须是布尔值
- `params["udp-relay-mode"]` / `params["congestion-controller"]` 如果存在，必须是非空字符串
- `params["request-timeout"]` 如果存在，必须是数字

### 4.7 `hysteria2`

当前常见字段：

- `credentials.password`
- `params.sni`
- `params.obfs`
- `params["obfs-password"]`
- `params.alpn`
- `params.pinSHA256`
- `params.insecure`
- `params.network`
- `params.mport`
- `params["hop-interval"]`
- `params.up`
- `params.down`
- `params.upmbps`
- `params.downmbps`

当前限制：

- `credentials.password` 必填
- `params.obfs` 如果存在，目前只支持 `"salamander"`
- `params["obfs-password"]` 只有在同时提供 `params.obfs` 时才合法
- `params.insecure` 必须是布尔值
- `params.pinSHA256` 必须是字符串或非空字符串数组
- `params.alpn` 必须是字符串或非空字符串数组
- `params.network` 如果存在，当前仅支持 `tcp` / `udp`
- 分享链接里的未知 query 参数会被直接视为 unsupported
- authority 里的多端口写法已支持，会收敛到 `port + params.mport`

## 5. 分享链接导入到字段的对应关系

### 5.1 导入阶段会做什么

当前解析器支持：

- `vless://`
- `trojan://`
- `vmess://`
- `ss://`
- `ssr://`
- `tuic://`
- `hysteria2://`
- `hy2://`

解析后每条节点都会被规范化成统一对象：

```json
{
  "name": "example",
  "protocol": "hysteria2",
  "server": "node.example.com",
  "port": 443,
  "credentials": {
    "password": "replace-me"
  },
  "params": {
    "sni": "sub.example.com"
  }
}
```

### 5.2 `ss` 导入字段 -> 最终 metadata 对照

| 导入位置 | 示例 | 最终落点 | 当前说明 |
| --- | --- | --- | --- |
| userinfo `cipher` | `aes-256-gcm` | `credentials.cipher` | 必填 |
| userinfo `password` | `replace-me` | `credentials.password` | 必填 |
| query `plugin` | `plugin=v2ray-plugin` | `params.plugin` | 当前只支持非空字符串 |
| fragment `#name` | `#HK-SS` | `name` | 不填时回退到 hostname |
| hostname | `hk.example.com` | `server` | 会先 decode |
| port | `:8388` | `port` | 必填，必须是合法端口 |

补充说明：

- `ss://` 既支持直接 userinfo，也支持整段 authority 的 Base64 变体
- `params.plugin-opts`、`params.pluginOpts`、`params.plugins` 当前不会自动结构化落库
- 如果你拿到的是复杂 plugin 变体，建议直接核对原始 JSON

### 5.3 `ssr` 导入字段 -> 最终 metadata 对照

| 导入位置 | 示例 | 最终落点 | 当前说明 |
| --- | --- | --- | --- |
| base64 body `server:port` | `ssr.example.com:443` | `server` / `port` | 必填 |
| base64 body `method` | `aes-256-cfb` | `credentials.cipher` | 必填 |
| base64 body `protocol` | `auth_aes128_md5` | `credentials.protocol` | 必填 |
| base64 body `obfs` | `tls1.2_ticket_auth` | `credentials.obfs` | 必填 |
| base64 body `password` | `passw0rd` | `credentials.password` | base64 解码后保留 |
| query `remarks` | `SSR Node` | `name` | 不填时回退到 hostname |
| query `protoparam` | `100:replace-me` | `params["protocol-param"]` | 可选 |
| query `obfsparam` | `sub.example.com` | `params["obfs-param"]` | 可选 |

补充说明：

- `ssr://` 当前已覆盖分享链接导入、结构化向导与统一托管输出
- `group` 这类更少见字段仍建议继续直接核对原始 JSON

### 5.4 `tuic` 导入字段 -> 最终 metadata 对照

| 导入位置 | 示例 | 最终落点 | 当前说明 |
| --- | --- | --- | --- |
| URL userinfo `uuid` | `1111...` | `credentials.uuid` | 必填 |
| URL userinfo `password` | `replace-me` | `credentials.password` | 必填 |
| hostname | `tuic.example.com` | `server` | 必填 |
| port | `:443` | `port` | 必填 |
| query `sni` | `sni=sub.example.com` | `params.sni` | 可选 |
| query `alpn` | `alpn=h3` | `params.alpn` | 当前按字符串或字符串数组保留 |
| query `congestion_control` | `congestion_control=bbr` | `params["congestion-controller"]` | 可选 |
| query `udp_relay_mode` | `udp_relay_mode=native` | `params["udp-relay-mode"]` | 可选 |
| query `disable_sni` | `disable_sni=1` | `params["disable-sni"]` | 解析为布尔值 |
| query `heartbeat` | `heartbeat=10s` | `params.heartbeat` | 可选 |
| query `request_timeout` | `request_timeout=8000` | `params["request-timeout"]` | 当前按数字保留 |
| query `zero_rtt_handshake` | `zero_rtt_handshake=1` | `params["reduce-rtt"]` | 统一收敛为布尔值 |
| fragment `#name` | `#TUIC Node` | `name` | 不填时回退到 hostname |

补充说明：

- `tuic://` 当前已覆盖分享链接导入、结构化向导与统一托管输出
- 完整配置导入时，`sing-box` 的 `tls.insecure` 与 Mihomo / Clash 的 `skip-cert-verify` 会统一收敛到 `params.insecure`
- 更少见的传输细节仍建议继续直接核对原始 JSON

### 5.5 `hysteria2` / `hy2` 导入字段 -> 最终 metadata 对照

| 导入位置 | 示例 | 最终落点 | 当前说明 |
| --- | --- | --- | --- |
| URL userinfo | `pass` 或 `user:pass` | `credentials.password` | userinfo 会组合成一个字符串 |
| hostname | `node.example.com` | `server` | 必填 |
| port | `:443` | `port` | 不填时默认按 `443` |
| authority 多端口 | `:8443,9443` / `:2000-3000` | `port` + `params.mport` | `port` 取首个端口；完整表达式保留到 `params.mport` |
| query `sni` | `sni=cdn.example.com` | `params.sni` | 可选 |
| query `obfs` | `obfs=salamander` | `params.obfs` | 当前只支持 `salamander` |
| query `obfs-password` | `obfs-password=secret` | `params["obfs-password"]` | 只有同时提供 `obfs` 才合法 |
| query `pinSHA256` | `pinSHA256=abc` | `params.pinSHA256` | 单值保留字符串；重复 query 会收敛成字符串数组 |
| query `alpn` | `alpn=h3` | `params.alpn` | 单值保留字符串；重复 query 会收敛成字符串数组 |
| query `mport` | `mport=2000-3000` | `params.mport` | 显式 query 优先；未提供时才回退到 authority 多端口原文 |
| query `hop-interval` | `hop-interval=30s` | `params["hop-interval"]` | 可选 |
| query `up` / `down` | `up=100` | `params.up` / `params.down` | 当前按字符串保留 |
| query `upmbps` / `downmbps` | `upmbps=100` | `params.upmbps` / `params.downmbps` | 当前按字符串保留 |
| query `insecure=true` | `insecure=true` | `params.insecure = true` | 只有解析为 true 才写入 |
| fragment `#name` | `#HK-HY2` | `name` | 不填时回退到 hostname |

补充说明：

- `hy2://` 会统一归一化为 `hysteria2`
- authority 里的多端口写法已支持，会收敛成 `port + params.mport`
- 如果 query 里也给了 `mport`，则优先保留 query 值
- 完整 `sing-box` 配置导入时，`network` / `server_ports` / `hop_interval` / `up_mbps` / `down_mbps` 与 `tls.certificate_public_key_sha256` 也会统一收敛到 `params.network` / `params.mport` / `params["hop-interval"]` / `params.upmbps` / `params.downmbps` / `params.pinSHA256`
- 未知 query 参数会直接报 unsupported

### 5.6 Base64 订阅文本会先被解包

如果你粘贴的是整段 Base64 包装订阅文本，系统会先尝试解包，再按每条分享链接继续解析。

### 5.7 导入成功不等于可直接输出

即使分享链接已经成功解析成节点，仍然要继续确认：

1. metadata 是否完整
2. 节点是否启用
3. 是否绑定到目标用户
4. 是否存在启用中的默认模板

## 6. 结构化向导当前覆盖范围

当前 Web 节点页已对这些协议提供常见字段向导：

- `vless`
- `trojan`
- `vmess`
- `ss`
- `ssr`
- `tuic`
- `hysteria2`

可以把它理解成“常见字段快速编辑器”，不是“完整协议建模器”。

特别是下面两类情况，仍建议直接核对原始 JSON：

1. `ss` 的复杂 plugin 变体
2. `ssr` / `tuic` 的更少见传输或扩展字段
3. `hysteria2` 的更多复杂链接组合变体

## 7. 写接口边界

与字段最相关的当前接口约束如下：

- `POST /api/nodes` 创建节点时必须提供 `name`、`protocol`、`server`、`port`
- `PATCH /api/nodes/:id` 允许通过 `credentials: null` / `params: null` 显式清空 metadata
- `sourceType=remote` 当前直接拒绝
- `sourceId` 当前不允许写给手动节点

如果你需要接口层的完整请求示例，看：

- `docs/API参考与接口约定.md`
