# SubForge 第十四批自动化验证执行清单

## 1. 文档目的

本文档用于把第十四批工作范围固定为“节点管理可用性补强”，优先解决当前 Web 节点表单信息不足、用户误以为已支持导入、以及节点 metadata 无法从前端录入的问题。

本轮只做三类工作：

1. 补节点使用说明文档，写清当前手动录入能力与未实现边界
2. 升级 Web 节点表单，支持 `credentials` / `params` 手动录入与更明确提示
3. 补节点 create metadata 回归测试，并同步状态文档

当前进展：`done`

- 已补节点使用说明文档，明确当前只支持手动录入节点
- 已补 Web 节点表单中的 `credentials` / `params` JSON 录入
- 已补 `POST /api/nodes` 接受 metadata 对象的请求级回归

## 2. 本轮边界

### 2.1 本轮要补

- `docs/节点管理与订阅使用说明.md`
- `apps/web/src/App.tsx`
- `apps/web/src/api.ts`
- `apps/web/src/styles.css`

### 2.2 本轮不补

- 订阅链接解析导入
- 远程节点源同步
- Web 组件测试框架
- 完整协议字段向导
- 节点测速或可用性探测

## 3. 具体执行项

### 3.1 节点文档补足

至少写清：

- 当前仅支持手动录入节点
- 节点创建后必须再绑定给用户
- 真实可用节点通常需要 `credentials` / `params`
- 当前“导入链接 / 远程节点源”仍未实现

### 3.2 节点表单升级

至少补齐：

- 创建节点时可录入 `credentials` JSON
- 创建节点时可录入 `params` JSON
- 编辑节点时可查看并更新 metadata
- 页面上明确提示当前不支持导入链接与远程节点源

### 3.3 自动化验证与状态同步

至少覆盖：

- `POST /api/nodes` 接受合法的 `credentials` / `params` 对象
- 节点 metadata 写入后能在响应与持久化层看到
- README 与状态文档同步到“节点管理可用性已补强”的描述

满足以下条件即可认为本轮完成：
