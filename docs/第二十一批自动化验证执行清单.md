# SubForge 第二十一批自动化验证执行清单

## 1. 文档目的

本文档用于把第二十一批工作范围固定为“`ss` 向导与 `hysteria2` 导入文档细化”，优先解决当前实现虽然已经补到 `ss` 结构化向导和 `hysteria2://` / `hy2://` 导入，但文档里对“实际回填哪些字段、哪些变体仍未覆盖、出问题时该怎么看”说明还不够具体的问题。

1. 为 README、节点使用说明、协议矩阵补 `ss` / `hysteria2` 的细粒度说明
2. 为已知问题、自动化验证与路线图补当前边界和后续口径

## 3. 具体执行项

### 3.1 使用说明细化

至少补齐：

- `ss` 协议向导当前实际覆盖字段：`cipher`、`password`、`plugin`
- `hysteria2://` / `hy2://` 当前导入会把哪些字段回填到 `credentials` / `params`
- `hysteria2://` 未显式端口时默认按 `443` 处理
- `hysteria2://` 多端口 authority 当前仍未支持

### 3.2 状态写实化

至少补齐：

- README 不再只写“支持了”，而是写清“支持到哪一层、还缺什么”
- 协议矩阵增加细粒度限制，避免把 `ss` / `hysteria2` 写成同等成熟
- 已知问题与路线图同步到同一套后续顺序

### 3.3 自动化校验

至少补齐：

- `scripts/smoke-check.mjs` 断言第二十一批文档存在
- smoke 断言覆盖 `ss` / `hysteria2` 的字段映射和多端口边界说明
