# SubForge 第十八批自动化验证执行清单

## 1. 文档目的

本文档用于把第十八批工作范围固定为“Base64 订阅文本解包补强”，优先解决当前虽然已经支持订阅 URL 远程抓取预览导入，但很多真实上游返回的是整段 Base64 包装订阅文本，导致后台抓取成功却没有解析出节点的问题。

本轮只做三类工作：

1. 为共享节点导入解析器补 Base64 订阅文本自动解包
2. 为 Worker 与 Web 补内容编码提示与对应回归
3. 同步 README、节点使用说明、自动化验证与路线文档

当前进展：`done`

- 节点可用性探测

## 3. 具体执行项

### 3.1 解析器补强

至少补齐：

- 当输入本身已经是逐行 `vless://` / `trojan://` / `vmess://` 时，保持原语义不变
- 当输入看起来是整段 Base64 包装订阅文本时，自动解包后再逐行解析
- 为解析结果补充内容编码标记：
  - `plain_text`
  - `base64_text`
- 为解析结果补充解包后的有效行数

### 3.2 Worker / Web 展示

至少补齐：

- 远程预览返回中包含内容编码信息
- 节点页在本地粘贴导入和远程预览里都能提示当前解析的是明文还是 Base64 订阅文本
- 继续保持“仅做一次性预览导入，不是远程源持续同步”的边界提示

### 3.3 自动化验证

至少补齐：

- 纯逻辑测试覆盖 Base64 包装订阅文本解包
- Worker 远程预览回归覆盖 Base64 订阅响应
- smoke 文档断言同步到第十八批

## 4. 完成标准

完成后应满足：

- 节点页既支持明文分享链接，也支持整段 Base64 订阅文本导入
- 订阅 URL 远程预览遇到 Base64 响应时可以自动解包并展示结果
- `npm run typecheck`
- `npm run build`
- `npm run test:smoke`
- `npm test`

---

## 5. 风险提示

本轮仍保留以下现实限制：

- 仅自动解包“Base64 包装的分享链接文本”，不是完整 Clash / sing-box 配置解析
- 不会自动识别所有上游订阅私有格式
- 即使能成功解包，节点参数仍建议人工核对
