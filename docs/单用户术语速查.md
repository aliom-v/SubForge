# SubForge 单用户术语速查

## 1. 这页是干什么的

这页只做一件事：

把当前单用户主界面里最容易混淆的按钮名、状态名和后台概念收成一份速查表。

如果你想看完整使用顺序，先看：

- `docs/单用户主链路速查.md`
- `docs/节点管理与订阅使用说明.md`

## 2. 最常见术语

| 术语 | 当前真实含义 | 什么时候需要关心 |
| --- | --- | --- |
| `自动同步源` | 持久化保存的上游订阅 URL，对应后端长期同步对象 `remote_subscription_sources` | 想让系统以后继续按 Cron 自动拉取同一个上游时 |
| `订阅 URL 解析` | 对某个上游 URL 做一次抓取和解析预览 | 还没决定是否要长期跟这个上游时 |
| `抓取并导入节点` | 把本次抓取结果作为一次性快照导入节点列表 | 只想先导进来，不想建立长期同步关系时 |
| `保存为自动同步源` | 保存上游 URL，并立刻同步一次，后续继续定时拉取 | 需要长期跟上游更新时 |
| `使用当前启用节点生成托管 URL` | 读取当前启用节点，刷新内部托管绑定，并更新 `/s/:token/mihomo` 与 `/s/:token/singbox` | 每次改完节点后，都要靠这一步刷新客户端实际拉到的内容 |
| `重置当前托管链接` | 轮换公开订阅 token，让旧 `/s/:token/...` 地址失效 | 怀疑链接泄漏，或想强制客户端换新地址时 |
| `当前启用节点` | 节点列表里现在被标记为启用的节点集合 | 想确认下一次生成会带哪些节点时 |
| `当前已托管绑定` | 上一次成功生成托管 URL 时真正写入托管订阅的节点集合 | 页面提示“需要重新生成”时 |
| `上游代理` / `upstreamProxy` | 节点链式代理关系的统一字段 | 配置 Mihomo `dialer-proxy` 或 sing-box `detour` 时 |
| `链式代理拓扑` | 前端用来展示和校验节点链路的图形区域 | 想排查缺失上游、坏引用、循环链路时 |
| `默认模板` / `托管骨架` | 系统维护的托管输出结构，用来承载规则、组和最终节点注入 | 你导入完整配置后，想知道为什么不是原样托管时 |
| `预览缓存` | 管理端预览和检查结果的缓存 | 页面看起来没刷新、怀疑还是旧预览时 |
| `公开订阅缓存` | `/s/:token/...` 公开订阅输出缓存 | 客户端拉到旧内容、需要结合生成时间一起判断时 |
| `内部审计` | 后端保留的审计记录对象，不是当前单用户主界面的独立操作入口 | 一般不用管，主要用于内部追踪和诊断 |

## 3. 三组最容易混淆的概念

### 3.1 `导入` 和 `生成`

- `导入`
  - 只是在改节点列表
- `生成`
  - 才是在刷新客户端真正拉取的托管订阅

### 3.2 `订阅 URL 解析` 和 `自动同步源`

- `订阅 URL 解析`
  - 先试抓一次，看能不能解析
- `自动同步源`
  - 真正保存下来，后续继续定时同步

### 3.3 `当前启用节点` 和 `当前已托管绑定`

- `当前启用节点`
  - 你现在打算输出什么
- `当前已托管绑定`
  - 客户端此刻实际拿到什么

两者不一致时，通常只需要重新执行一次：

- `使用当前启用节点生成托管 URL`
