# SubForge 第二批自动化验证执行清单

## 1. 文档目的

本文档用于把第二批测试范围固定下来，避免“继续补测试”变成无边界扩散。

本轮只做三类回归测试：

1. 缓存失效 helper
2. 管理员鉴权语义
3. 审计日志脱敏

当前进展：`done`

- 已补缓存失效 helper 回归测试
- 已补管理员登录 / 鉴权关键路径测试
- 已补审计脱敏回归测试
- 已修复非法 Bearer token 会触发异常而不是返回 401 的问题

---

## 2. 本轮边界

### 2.1 本轮要补

- `apps/worker/src/cache.ts` 的缓存失效 helper 行为
- `apps/worker/src/index.ts` 的管理员登录 / 鉴权关键分支
- `apps/worker/src/audit.ts` 与审计写入链路的脱敏行为

### 2.2 本轮不补

- 真实 D1 / KV / Cron 集成测试
- Web 组件测试
- 规则源同步端到端网络测试
- 完整权限矩阵测试
- 覆盖率门槛

---

## 3. 具体执行项

### 3.1 缓存失效 helper

目标文件：

- `apps/worker/src/cache.ts`

本轮至少覆盖：

- `invalidateUserCaches` 默认会清理同一用户的 `mihomo` / `singbox` 公开订阅缓存和预览缓存
- `invalidateUserCaches` 传入目标类型时只清理指定 target
- `invalidateAllUserCaches` 会基于 `listUserCacheRefs` 返回结果批量失效
- `invalidateNodeAffectedCaches` 会基于 `listUsersByNodeId` 返回结果批量失效

### 3.2 管理员鉴权语义

目标文件：

- `apps/worker/src/index.ts`
- `apps/worker/src/security.ts`

本轮至少覆盖：

- 登录成功后返回管理员会话 token
- 错误密码返回 401
- 禁用管理员登录返回 403
- 缺失 Bearer token 访问受保护接口返回 401
- 非法 Bearer token 访问受保护接口返回 401
- 合法 token 访问 `/api/admin/me` 返回当前管理员信息
- 禁用管理员即使 token 合法，访问受保护接口也返回 403

### 3.3 审计日志脱敏

目标文件：

- `apps/worker/src/audit.ts`
- `apps/worker/src/index.ts`
- `apps/worker/src/repository.ts`

本轮至少覆盖：

- `sanitizeAuditPayload` 会对 `token` / `password` / `secret` / `authorization` 相关字段脱敏
- 嵌套对象与数组中的敏感字段也会被脱敏
- 审计日志列表接口返回的历史 payload 会再次做脱敏，避免旧数据直出

## 4. 实现方式

本轮继续使用当前仓库已经落地的最小测试方案：

- `node:test`
- 手写 mock
- `tests/helpers/load-ts-module.mjs` 动态加载 TypeScript 源文件

约束：

- 不新增大型测试框架
- 不要求把 Worker 全量路由都覆盖
- 只补最近已经修过、最容易回归的关键语义
