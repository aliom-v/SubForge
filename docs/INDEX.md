# SubForge 文档导航

## 1. 单人使用只看这几篇

如果你只想按当前单用户托管模式使用，建议按下面顺序阅读：

1. `README.md`
2. `docs/单用户托管模式需求.md`
3. `docs/节点管理与订阅使用说明.md`
4. `docs/部署指南.md`

这 4 篇已经覆盖：

- 当前产品目标与边界
- 唯一主流程
- 三种导入入口
- 统一生成托管 URL
- 部署、更新与最小排障

如果你只是自己部署自己用，其他文档都可以先忽略。

## 2. 内部参考

下面这些文档默认属于仓库内部资料，主要服务于开发、接口联调、运维、发布和长期维护，不是单人使用主线。

### 2.1 使用补充

- `docs/节点字段字典.md`
  - 协议字段该落到 `credentials` 还是 `params`
- `docs/节点协议示例库.md`
  - 可直接复制的协议样例
- `docs/排障与常见问题.md`
  - 症状驱动 FAQ：本地开发、部署异常、缓存、鉴权、`429`、HTML 回退

### 2.2 开发与接口

- `docs/API参考与接口约定.md`
  - 面向开发联调的路由、成功/失败包络、请求示例与 OpenAPI 入口
- `docs/API错误码与响应头说明.md`
  - 公共协议约定、Bearer 鉴权、错误码、缓存头、限流头、高频报错与成功语义误区
- `openapi.yaml`
  - 机器可读的正式 OpenAPI 规范，可用于 Swagger / Redoc / 代码生成，并已补关键 request / response examples
- `npm run test:contract`
  - 纯 Node.js 的契约检查，校验公开鉴权边界与前端路由清单是否仍与规范一致

### 2.3 架构、数据与实现

- `docs/实施方案.md`
  - 长期设计基线、模块边界与架构原则

### 2.4 运维与发布

## 3. 维护约定

- `README.md` 只保留项目总览、快速开始和单用户主路径，不重复展开长篇 SOP
- `docs/单用户托管模式需求.md` 只维护当前产品目标、范围与非目标
- `docs/节点管理与订阅使用说明.md` 只维护三种导入入口、托管 URL 和当前限制
- `docs/部署指南.md` 只保留单人部署主路径、绑定、最小验收与首轮排障
- `docs/高级部署与数据维护.md` 承担 GitHub Actions、D1 备份恢复与对象存储归档细节
- 其余文档默认都属于内部参考，不再作为单人使用入口反复强调
- API 文档按“人工参考 / 协议与错误 / 正式 OpenAPI”三层维护，避免同一信息四处复制

## 4. 清理约定

- 根目录 `docs/` 只保留仍在主动维护的文档
- 已经合并掉的旧专题，不再继续保留兼容占位页
- 需要追溯历史收敛过程时，统一到 `docs/archive/` 查看批次记录
