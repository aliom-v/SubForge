# SubForge 第三批自动化验证执行清单

## 1. 文档目的

本文档用于把第三批测试范围固定为“同步流程与配置变更导致的缓存失效”，避免继续扩展到更重的集成环境。

本轮只做四类验证：

1. 模板默认切换导致的缓存失效
2. 当前生效模板更新导致的缓存失效
3. 规则源启停导致的缓存失效
4. 规则源同步成功 / 跳过时的缓存失效语义

当前进展：`done`

- 已补模板默认切换与当前生效模板更新的缓存失效测试
- 已补规则源启停变更的缓存失效测试
- 已补规则源同步成功 / 跳过，以及仅同步启用规则源的测试

---

## 2. 本轮边界

### 2.1 本轮要补

- `apps/worker/src/index.ts` 中模板与规则源相关 handler 的失效行为
- `apps/worker/src/sync.ts` 中规则源同步成功 / 跳过时的失效行为

### 2.2 本轮不补

- 真实远程网络集成测试
- Web 页面级测试
- 全量 Worker 路由测试
- Cron 调度端到端测试
- 覆盖率门槛

---

## 3. 具体执行项

### 3.1 模板相关缓存失效

目标文件：

- `apps/worker/src/index.ts`

本轮至少覆盖：

- `POST /api/templates/:id/set-default` 在默认模板发生切换时，会按目标类型失效用户缓存
- `PATCH /api/templates/:id` 更新当前生效模板内容时，会按目标类型失效用户缓存
- 非当前生效模板的普通更新，不应误触发目标缓存失效

### 3.2 规则源启停导致的缓存失效

目标文件：

- `apps/worker/src/index.ts`

本轮至少覆盖：

- `PATCH /api/rule-sources/:id` 当 `enabled` 状态发生变化时，会失效全部用户缓存
- 只修改规则源名称或 URL 等非启停字段时，不应误触发全量缓存失效

### 3.3 规则源同步导致的缓存失效

目标文件：

- `apps/worker/src/sync.ts`

本轮至少覆盖：

- `syncRuleSourceNow` 在快照内容发生变化时，会写入快照、记录同步结果并失效全部用户缓存
- `syncRuleSourceNow` 在内容未变化时返回 `skipped`，且不应触发缓存失效
- `runEnabledRuleSourceSync` 只会处理启用的规则源

- `node:test`
- `tests/helpers/load-ts-module.mjs`

- 不引入新的测试框架
- 不做真实 D1 / KV / 远程 HTTP 依赖
